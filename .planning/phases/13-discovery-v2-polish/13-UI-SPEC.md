## Typography

DESIGN.md defines the type scale; Phase 13 uses 4 sizes and exactly 2 weights. All numerics consume the `font-metric` utility (Geist Mono + tabular-nums); all UI text consumes the body sans (DM Sans).

**Weight inventory (must equal 2):**
- **400 (regular)** — body copy, table cells, numerics (Geist Mono is visually distinct from DM Sans by typeface alone; no medium needed for the numeric row)
- **600 (semibold)** — labels, headings, badges, micro-uppercase tracked text

| Role | Family | Size | Weight | Line Height | Phase 13 usage |
|------|--------|------|--------|-------------|----------------|
| Body | DM Sans | 14px | 400 (regular) | 1.5 | Drawer descriptions, search placeholder, empty-state copy, table body cells |
| Label | DM Sans | 12px | 600 (semibold) | 1.4 | Drawer section headers (`text-xs font-semibold uppercase tracking-wider` already in code at `StrategyFilters.tsx:154`); count-badge digit (see Micro row below) |
| Heading | DM Sans | 18px | 600 (semibold) | 1.3 | Customize drawer title (`text-lg font-semibold` matches the All-Filters drawer heading at `StrategyFilters.tsx:408`) |
| Numeric | Geist Mono | 14px | 400 (regular) | 1.4 | Sparkline-adjacent percentages, watchlist count badge digit when ≥2 chars (e.g. "12"). Geist Mono's tabular glyphs and tighter letterforms carry visual weight without needing a medium grade. |
| Micro | DM Sans | 11px | 600 (semibold) | 1.2 | Watchlist tab count badge, All-Filters active-count chip. See "Micro-label exception" below — phase 13 promotes the existing 10px/font-bold chip to 11px/semibold to land inside the 2-weight contract while keeping the white-on-accent legibility floor (DESIGN.md "Micro: 10–11px (labels, badges, uppercase tracking)"). |

**Micro-label exception (size promotion to absorb weight constraint):** The existing `activeCount` chip at `StrategyFilters.tsx:319` ships as `text-[10px] font-bold` (700) on `bg-accent text-white`. To collapse to 2 weights without losing white-on-#1B6B5A legibility at small size, Phase 13 promotes the badge size from 10px to 11px and the weight from 700 to 600. Both the new Watchlist count badge AND any subsequent visual touch to the existing All-Filters chip MUST adopt this `text-[11px] font-semibold` pair. DESIGN.md "Micro: 10–11px" allows 11px explicitly; this is a reuse, not a new size tier.

**Heading line-height** = 1.3 (per DESIGN.md "Hero / H2 / H3" rhythm; comfortable not display).
**Body line-height** = 1.5 (DESIGN.md default).

No Display tier (Instrument Serif) is used on `/discovery/[slug]`.

**Total declared weights:** 2 — `400 (regular)` and `600 (semibold)`. No `font-medium` (500), no `font-bold` (700) anywhere in Phase 13 surfaces.

| Role | Token | Hex | Phase 13 usage |

| Dominant (60%) — page background | `--color-page` | #F8F9FA | Discovery page wrapper, sticky filter-row backdrop (`bg-page` at `StrategyFilters.tsx:300`), drawer scrim |

| Destructive | `--color-negative` | #DC2626 | Sparkline stroke when `final < 0`. NOT used as button background in Phase 13 (no destructive actions ship — see Copywriting Contract). |
| Positive | `--color-positive` | #16A34A | NOT used on sparkline strokes in Phase 13 (DESIGN.md DIFF-05: single accent for trace). Only retained on existing return-percentage cells via `metricColor()` helper at `lib/utils.ts`. |
| Benchmark grey | `--color-chart-benchmark` | #94A3B8 | Sparkline stroke when `final == 0`. Already the documented sparkline neutral. |
| Border | `--color-border` | #E2E8F0 | All 1px dividers (table rows, filter row bottom, drawer borders, segmented-tab outline) |
| Text primary | `--color-text-primary` | #1A1A2E | Active tab label, drawer heading, strategy-name link rest state |
| Text secondary | `--color-text-secondary` | #4A5568 | Inactive tab label, drawer body copy, "Hide examples" label |
| Text muted | `--color-text-muted` | #718096 | Search placeholder, empty-state body, "X results" pagination caption, count-badge text inside accent fill (overridden to white — see below) |

**Accent reserved for** (and ONLY for):
1. **Active state of the All / My Watchlist segmented tab** — active tab uses `bg-accent/10 text-accent` (matches the existing view-mode toggle pattern at `StrategyFilters.tsx:372`); inactive tab uses `text-text-secondary`, no fill, hover lifts to `hover:bg-page`.
2. **Filled star icon** when a strategy is on the user's watchlist — fill = `var(--color-accent)`, stroke none. Outline (unstarred) state uses `stroke="currentColor"` over `text-text-muted`, no fill.
3. **Watchlist count badge** — 18×18 pill with `bg-accent text-white text-[11px] font-semibold` (re-uses the existing All-Filters active-count chip pattern at `StrategyFilters.tsx:319`, promoted to 11px/600 per the Typography "Micro-label exception").
4. **Sparkline stroke + 1px end-point dot when `final > 0`** — `stroke="var(--color-accent)"` (this is already `--color-chart-strategy: #1B6B5A`, alias of accent).
5. **Active form control focus rings** inside the Customize drawer (`focus-visible:ring-accent` — matches existing input convention at `globals.css:18`).
6. **"Save preferences" primary CTA in the Customize drawer footer** — re-uses `<Button variant="primary">` which is `bg-accent text-white hover:bg-accent-hover` (exact token chain at `Button.tsx:9`).

**Never use accent for:**
- Sparkline strokes when `final < 0` or `final == 0` (DIFF-05: sign-driven single-color rule below).
- Any text that is not on top of an accent surface (the sole accent-on-white text is the active-tab label).
- Decorative borders or generic interactive states beyond the six elements above.

**Sparkline color rule (DESIGN.md DIFF-05 — locked in CONTEXT.md):**

| Trace condition | Stroke | Fill (if `fill=true` shaded variant ever opted in) |
|-----------------|--------|-----------------------------------------------------|
| Final value > 0 | `#1B6B5A` (`--color-accent`) | Same hex at opacity 0.1 (matches existing `Sparkline.tsx:46` opacity) |
| Final value < 0 | `#DC2626` (`--color-negative`) | Same hex at opacity 0.1 |
| Final value == 0 | `#94A3B8` (`--color-chart-benchmark`) | Same hex at opacity 0.1 |

This rule is computed at the call site (`StrategyTable.tsx` table cell + `StrategyGrid.tsx` card body), not inside `Sparkline.tsx` — matches the codebase's "Sparkline is single-color, caller picks the color" contract at `Sparkline.tsx:14`.

The `sparkline_drawdown` cell that already exists in the table (`StrategyTable.tsx:319` passes `color="var(--color-negative)"`) is OUT OF SCOPE for the sign-driven rule — drawdown is by definition non-positive, so its color is statically negative. Phase 13 only re-colors `sparkline_returns`.

On failure, the icon reverts and an inline 14px text-negative `<span>` reads `Couldn't update watchlist. Retry?` next to the star for 4 seconds, then auto-dismisses. | Optimistic UI + visible reconciliation per CLAUDE.md core principle "Verification Before Completion". The "Retry?" word is a hint, not a button — clicking the star itself retries. |
| Error state — star toggle persistent failure (after 1 retry) | Inline below the strategy row (table mode) or below the card (grid mode): `Watchlist offline. **Destructive actions in Phase 13:** none. Removing a strategy from the watchlist is **not** destructive — it's a toggle whose inverse is a single star click. No confirmation modal. (Per CLAUDE.md "Minimal Impact" — adding a confirm flow for a 1-click reversible op is unnecessary friction.)

**Voice:** declarative, action-first, never apologetic. No emojis. No "Oops!" / "Whoops!" / exclamation points anywhere. Matches DESIGN.md "Industrial / Utilitarian — typography and data do all the work".

**Banned-label compliance:** No CTA in Phase 13 uses a generic banned label (`Save`, `Submit`, `OK`, `Click Here`, `Cancel`). Every CTA is verb-noun specific to its destination: `Save preferences`, `Reset to defaults`, `Close customize panel`, `Apply filters` (existing — out of phase 13 scope but noted for parity).

## Accessibility Contract

| Requirement | Phase 13 implementation |
|-------------|--------------------------|
| Focus order in filter row | left-to-right: Search input → All Filters button → All/Watchlist tabs (single tab stop, arrow-keys move within) → Hide-examples checkbox → Sort-key select → Sort-dir select → Settings-cog button → View-toggle group (single tab stop, arrow-keys move) |

### Visual Hierarchy (Discovery list eye-path)

The Discovery table/grid carries 8+ data columns per row; without an explicit hierarchy declaration the executor may weight every column equally and dilute the scan axis. Phase 13 declares the priority chain so layout decisions inherit a single rule:

1. **Primary visual anchor — strategy-name column (the scan axis).** Every row is read first by name. The name renders at body-14px DM Sans 600 (semibold) on `text-text-primary`. The leading star column sits to the *left* of the name to act as a stable left rail without competing for first-glance attention; the star is 16×16 at `text-text-muted` (rest), only lifting to accent when starred or hovered.
2. **Secondary anchor — sparkline cell (the data anchor).** Once the eye lands on a name, the next saccade is to the trend trace. The sparkline is the only chromatic surface on a default row (sign-driven: accent / negative / benchmark-grey), and its single-color rule keeps it identifiable as a unit rather than a decoration. Numeric metrics around the sparkline render at body-14px Geist Mono 400 — quieter than the name, louder than nothing.
3. **Tertiary affordance — star toggle (interaction handle).** The star is the only interactive element discoverable on hover; it lives in the leading column where the eye already passed and lifts to `text-accent` on row hover so it surfaces just-in-time without ever competing with the name or sparkline at rest.

In grid mode the same hierarchy applies, expressed differently: the Card's title is the scan axis, the embedded sparkline is the data anchor, and the top-right star is the affordance — physically separated from the title so the affordance never reads as part of the title block. The executor MUST NOT add a fourth tier (no row-level chevrons, no expansion carets, no avatar circles) without re-opening the contract.

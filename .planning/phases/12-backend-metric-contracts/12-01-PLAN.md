D-15 audit SQL (run as-is — exchange list is the locked scope):
```sql
SELECT exchange,
       COUNT(*) AS total,
       COUNT(*) FILTER (WHERE is_maker IS NOT NULL) AS populated,
       (COUNT(*) FILTER (WHERE is_maker IS NOT NULL))::float / NULLIF(COUNT(*), 0) AS coverage
FROM raw_fills
WHERE exchange IN ('binance', 'okx', 'bybit')
GROUP BY exchange
ORDER BY exchange;
```

D-15 branch logic (from CONTEXT.md):
- ALL 3 exchanges ≥ 99% coverage → ship 4-bucket Trade Mix (long_maker, long_taker, short_maker, short_taker)
- ANY 1 of 3 exchanges < 99% coverage → descope maker/taker dimension to v0.17.1, ship 2-bucket fallback (long, short)
- Deribit → N/A by design (document, do not query)

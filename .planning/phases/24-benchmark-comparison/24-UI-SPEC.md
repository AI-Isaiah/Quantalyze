# Phase 24 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** INCREMENTAL benchmark dimension on the EXISTING scenario projection surface (`ScenarioComposer.tsx`). Three additions — a BTC overlay on the projection equity chart, a "vs BTC" active-return metrics section (tracking error / information ratio / alpha / beta over the aligned window), and an honest "Benchmark comparison unavailable" empty state. Reuses existing tokens, components, and copy patterns. No new visual direction. Source of truth: `DESIGN.md` + `24-CONTEXT.md`.

## Color

Inherits the DESIGN.md restrained palette. No new color introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8F9FA` page / `#FFFFFF` surface | Composer background + the metrics-section / empty-state card surfaces |
| Secondary (30%) | `#E2E8F0` border, `#F1F5F9` chart track, `#4A5568` text-secondary, `#64748B`/`#94A3B8` text-muted | Hairline dividers, metric-row borders, label + value text, chart gridlines |
| Accent (10%) | `#1B6B5A` muted institutional teal | **Strategy** equity line ONLY (`CHART_ACCENT`, already the scenario series color). Reserved-for list below. |
| Benchmark line | `#94A3B8` (`CHART_TEXT_MUTED` / DESIGN.md "Chart benchmark") | The BTC overlay line — muted slate, 1px, visually subordinate to the accent strategy line. NEVER accent teal. |
| Destructive | `#DC2626` negative | **Not used in this phase.** Phase 24 has no errors, no destructive actions. A missing/insufficient benchmark is honest absence (neutral), NOT an error. |

**Accent (`#1B6B5A`) reserved for:** the scenario/strategy equity-curve line and (where a comparison leader is crowned in a reused table primitive) the winner ✓ — that is all. The benchmark overlay, all metric values, all empty-state copy, and all labels are NEUTRAL (text-muted / text-secondary). No accent on the benchmark line, no accent on the metrics section.

**Honesty color invariant (load-bearing):** the "Benchmark comparison unavailable" state is rendered via `EmptyStateCard` — a neutral muted card with **NO `role="alert"`**, **NO red/negative**, **NO amber/warning**. Insufficient/missing benchmark coverage is honest absence, not a failure. (Same contract as the Phase-21/22 below-threshold empty states and the `#509` fix.)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Toggle control (overlay show/hide) | **"BTC Benchmark"** with a muted `#94A3B8` line swatch — verbatim from `EquityCurve.tsx` `showBenchmark` control. Label "vs BTC" elsewhere; the chart legend/title for the overlay series is **"BTC"**. |
| Metrics-section heading | **"vs BTC over {N} overlapping days"** — `{N}` = the aligned intersection-window day count (NEVER the wider union window). |
| Metric row labels | **"Tracking Error"**, **"Information Ratio"**, **"Alpha"**, **"Beta"** (DM Sans label; values in Geist Mono). |
| Methodology disclosure | `methodologyLine(n)` → "Historical realized · {N} overlapping days · not a forecast." **plus** the note **"Metrics are 252-day annualized active returns."** (Reuse `methodologyLine` from `src/lib/scenario-history.ts`; the 252-note is a new literal appended in the same 11px `text-text-muted` caveat line.) |
| Empty state heading | **"Benchmark comparison unavailable"** (via `EmptyStateCard` `heading`). |
| Empty state body | Names the SPECIFIC reason — heading MUST match body (the `#509` lesson). Two cases: (a) **no overlap / window not covered** → "The BTC benchmark series doesn't cover this scenario's date window, so there's nothing to compare against. Pick strategies whose history overlaps the benchmark." (b) **overlap below the 30-day floor** → "These dates share {N} overlapping days with the BTC benchmark — fewer than the 30 needed for an honest comparison. Pick strategies with longer common history." NEVER conflate "no benchmark data" with "too few days". |
| Degenerate single-metric value | A null / non-finite TE / IR / alpha / beta renders em-dash **"—"** (via `formatPercent`/`formatNumber`). NEVER a fabricated `0` / `"0.00%"` / `"N/A"` / `"0.00"`. |
| Error state | **Not applicable.** Phase 24 surfaces no error envelope — a missing benchmark is the neutral empty state above, not `ErrorEnvelope`. The benchmark GET route's transport failures degrade to the same honest "unavailable" empty state, never a red alert on the projection. |
| Destructive confirmation | **None.** Phase 24 has no destructive actions (read-only overlay + computed metrics). |

**Honesty / no-invented-data invariant (milestone-load-bearing):** a missing or insufficient benchmark window MUST surface as the neutral "Benchmark comparison unavailable" `EmptyStateCard`, NEVER a mismatched-window comparison or a fabricated number. Alignment is the INTERSECTION (inner-join) of scenario dates ∩ benchmark dates — never the scenario's internal zero-filled date UNION. The 30-day benchmark floor uses `evaluateSampleFloor(n, 30)` (`src/lib/sample-floor.ts`).

## Component & Interaction Inventory

| Element | Reused primitive | Contract |
|---------|------------------|----------|
| BTC overlay on the projection equity chart | `EquityChart` (composer widget) wrapping `EquityCurve.tsx`'s `benchmarkSeries` + `showBenchmark` pattern | Benchmark line: `#94A3B8`, `lineWidth: 1`, title "BTC", same `PRICE_FORMATTER` ("{pct}%"). Subordinate to the 2px accent strategy line. Toggle defaults ON; label "BTC Benchmark" with muted swatch. The composer's `EquityChart` does not yet carry a `benchmarkSeries` prop — wiring it (or lifting the toggle to panel level like the v2 strategy panel) is the planner's call; the visual contract above is fixed. |
| Metrics section ("vs BTC") | New labeled section mounted in `ScenarioComposer.tsx` below the projection chart block (reuse `mt-6`/`mt-4` rhythm); metric rows mirror `ScenarioCompareTable` row styling (`text-xs` label `text-text-muted`, right-aligned `.font-metric` value `text-text-secondary`, hairline `border-b border-border/50` rows) | 4 rows: Tracking Error, Information Ratio, Alpha, Beta. Values Geist Mono tabular-nums. Layout (table vs definition-list vs strip) is the planner's call within these tokens. |
| Honest empty/degenerate state | `EmptyStateCard` (`src/components/ui/EmptyStateCard.tsx`) — verbatim | Heading "Benchmark comparison unavailable" + reason-specific body (see copywriting). Neutral card, no `role="alert"`, no red/amber. Replaces the metrics section AND suppresses/empties the overlay when the window is uncoverable or below floor. |
| Methodology stamp | `methodologyLine(n)` (`src/lib/scenario-history.ts`) | 11px `text-text-muted`, stamped under the metrics section; `{N}` = aligned intersection count; append "Metrics are 252-day annualized active returns." |

**A11y:** benchmark line color `#94A3B8` is decorative (the series is also labeled "BTC" in the legend + toggle text), so it does not rely on color alone. The toggle is a keyboard-operable `<input type="checkbox">` with a `<label>` (verbatim `EquityCurve` pattern). The empty state is informational text (no `role="alert"`; it is honest absence, not a blocking error). Recharts/lightweight-charts `accessibilityLayer` opt-out convention (DESIGN.md 2026-04-30) applies to any new chart series — but this phase only ADDS a series to an existing chart, no new chart root.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Admin RLS policy: `profiles.user_id` → `profiles.id`**

- **Found during:** Task 1 authoring
- **Issue:** Plan §5 (RLS policies) specifies the admin SELECT policy as
  `EXISTS (SELECT 1 FROM public.profiles p WHERE p.user_id = auth.uid() AND p.is_admin = true)`. The `profiles.user_id` column does not exist —
  `profiles.id` IS the user id (FK to `auth.users.id`). Applying the plan
  verbatim would fail the migration with "column profiles.user_id does not exist".
- **Fix:** Use the codebase-canonical pattern `EXISTS (SELECT 1 FROM public.profiles p WHERE p.id = auth.uid() AND p.is_admin = true)` — same pattern used in 8 prior migrations (`20260417031851_user_app_roles.sql`,
  `20260409133654_profile_pii_revoke_hardened.sql`, etc.).
- **Files modified:** `supabase/migrations/20260522120000_csv_daily_returns.sql` (policy `csv_daily_returns_admin_select`)
- **Commit:** `691c8fc4`

### Non-deviation reconciliations vs prior art

These are not deviations from the plan — they ARE the plan — but they
diverge from prior-art SHA `748b2d74` and warrant a note so a reviewer
comparing replan vs prior art does not mistake them for regressions:

- **Column type:** plan specifies `DOUBLE PRECISION` for `daily_return` (prior art used `NUMERIC`). The plan wins.
- **Composite PK on (strategy_id, date):** plan specifies this directly (prior art used `id UUID PRIMARY KEY` + `UNIQUE` constraint). The plan wins — both produce equivalent uniqueness guarantees, but the composite PK avoids carrying an unused surrogate id.
- **Policy names:** plan uses `csv_daily_returns_service_role_all` (prior art used `csv_daily_returns_service_all`). The plan wins.
- **Admin policy gate:** plan uses an explicit `EXISTS ... profiles ... is_admin = true` check (prior art used `current_user_has_app_role(ARRAY['admin'])`). The plan wins.
- **Upsert shape:** plan uses set-based `INSERT ... SELECT FROM jsonb_array_elements ... ON CONFLICT DO UPDATE` with `GET DIAGNOSTICS ROW_COUNT` (prior art used per-row FOR loop + manual counter). The plan wins — same idempotency, simpler control flow.
- **`updated_at` column:** plan adds it (prior art did not). The plan wins — useful for ops debugging "when was this row last touched by an upsert".

## Deviations from Plan

- **[Rule 2 — Auto-add missing critical functionality]** Extended `SyncProgress.toSyncStatus`'s polling forwarder to include `next === "complete_with_warnings"` in its `onStatusChange` gate.
  - **Found during:** Task 1
  - **Issue:** The plan's Task 1 only spelled out adding the new branch to the `switch`. The polling block at line ~191 of `SyncProgress.tsx` had a string-equality allow-list (`next === "computing" || next === "complete" || next === "error"`) that would silently drop the new `complete_with_warnings` value mid-poll, defeating the whole point of the union extension.
  - **Fix:** Added `next === "complete_with_warnings"` to the allow-list so the wizard chip transitions correctly.
  - **Files modified:** `src/components/strategy/SyncProgress.tsx`
  - **Commit:** `3647bcb0`
- **[Rule 2 — Auto-add missing critical functionality]** Added a `Test 8c (arity lock)` source-shape regex check on top of the plan's strict signature regex.
  - **Reason:** The plan's Test 8c (arity lock) called out using `unifiedCsvFinalizeHandler.length` at runtime, but the handler is module-private — there is no public export to reach `.length` from. Substituted an equivalent source-shape check that pins the structural arity at 1 (single-args opener + single-object literal at the only call site), which catches the same regression the plan was after.
  - **Files modified:** `src/__tests__/csv-validate-route.test.ts`
  - **Commit:** `e9477b53`
- **[Rule 3 — Auto-fix blocking]** Rewrote the strict signature-shape regex inside Test 8b to avoid the `/s` flag (which requires ES2018, but tsconfig targets ES2017).
  - **Issue:** `error TS1501: This regular expression flag is only available when targeting 'es2018' or later.`
  - **Fix:** Switched to a portable `[\s\S]` pre-match that captures the whole signature block, then ran the field-presence check against that block.
  - **Files modified:** `src/__tests__/csv-validate-route.test.ts`
  - **Commit:** `e9477b53`

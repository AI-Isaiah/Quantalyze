<objective>
Extend the Python analytics-service to compute CSV-sourced analytics via the same `compute_all_metrics` pipeline as exchange-backed strategies:

1. `csv_validator.py` — extend `validate()` envelope to return `daily_returns_series` for both `daily_returns` (verbatim) and `daily_nav` (via `pct_change().dropna()`). Trades format omits the field.
2. `analytics_runner.py` — add `run_csv_strategy_analytics(strategy_id)` parallel to existing `run_strategy_analytics`. Routes directly through `compute_all_metrics` — NO trades/fills/positions chain. Sets `data_quality_flags.csv_source = True`. Logs warning before re-raise in `_mark_unrecoverable` (PR #273).
3. `job_worker.py` — register `compute_analytics_from_csv` in `TIMEOUT_PER_KIND` (10 min) + dispatch table; new handler `run_compute_analytics_from_csv_job`.
4. `main_worker.py` — register `WATCHDOG_PER_KIND_OVERRIDES["compute_analytics_from_csv"] = "15 minutes"` (mirrors compute_portfolio; satisfies `test_every_kind_has_watchdog_headroom`).

Purpose: Without this code, the Railway worker has no handler for the new job kind and the `csv_daily_returns` rows never get processed.

Output: Code + tests on disk. No live deploy by this plan — Plan 07 ships analytics-service to Railway.
</objective>

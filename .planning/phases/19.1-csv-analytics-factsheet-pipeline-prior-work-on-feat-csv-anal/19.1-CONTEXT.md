## Phase Boundary

Replace the v0.24.6.0 `analyticsMissingMessage` stop-gap (which tells CSV-uploaded
users "platform-computed analytics are not generated for CSV-uploaded strategies in
this release") with the real CSV → analytics → factsheet pipeline.

CSV-uploaded strategies must produce CAGR / Sharpe / drawdown / sparkline metrics
through the same `compute_all_metrics` path as exchange-backed strategies — fed from
a persisted `csv_daily_returns` table rather than a synthesized trades → fills →
positions chain.

<decisions>
## Implementation Decisions

### DB Foundation (Tasks 1-2)

- New table `csv_daily_returns(strategy_id, date, daily_return)` with `UNIQUE
  (strategy_id, date)`. UNIQUE constraint's implicit B-tree serves both the
  worker SELECT and the ON CONFLICT upsert — no explicit secondary index
  (migration-reviewer caught the redundant index in PR #272; do not re-introduce).
- `persist_csv_daily_returns(p_user_id UUID, p_strategy_id UUID, p_rows JSONB)`
  RPC, `SECURITY DEFINER`, 3-tier RLS (owner only). Pre-hardened per #272:
  - Collapse missing-strategy (22023) and wrong-owner (42501) into a single
    42501 to close the probe-oracle.
  - `jsonb_typeof(p_rows) <> 'array'` → 22023 before `jsonb_array_length`.
  - GRANT TO authenticated (matches `finalize_csv_strategy` migration 093 pattern
    — narrowing to service_role would NULL `auth.uid()` and break the function).
    Inline comment justifies the GRANT shape so future reviewers don't re-litigate.
- Register `compute_analytics_from_csv` in `compute_jobs_kind_target_coherence`
  as a strategy-scoped kind (extends the existing `kind IN ('sync_trades',
  'compute_analytics', ...)` arm). Strict superset of the current constraint —
  no in-flight insert can fail under the swap.
- Migration timestamps must be `> 20260521210000` (latest on main). Use
  `20260522nnnnnn` format. Two migrations: `csv_daily_returns.sql` and
  `compute_analytics_from_csv_kind.sql`.

### Python analytics-service (Tasks 4-6)

- `csv_validator.py` `validate` envelope returns `daily_returns_series` for both
  `daily_returns` (verbatim) and `daily_nav` (via `pct_change().dropna()`).
  Trades format omits the field.
- `analytics_runner.py` ships `run_csv_strategy_analytics(strategy_id)` parallel
  to the existing `run_strategy_analytics`. Sets `data_quality_flags.csv_source
  = True`; handles benchmark unavailability (`benchmark_unavailable=True` +
  `benchmark_note`); routes directly through `compute_all_metrics` — NO
  trades / fills / positions chain. Per #273, log a warning before re-raise on
  the `_mark_unrecoverable` silent-swallow path so a stuck `computing` row has
  log evidence.
- `job_worker.py` dispatch branch for `compute_analytics_from_csv`. Add to:
  - `TIMEOUT_PER_KIND` = 10 minutes
  - `WATCHDOG_PER_KIND_OVERRIDES` = 15 minutes (mirrors `compute_portfolio`)
    — the existing `test_every_kind_has_watchdog_headroom` invariant requires
    watchdog > handler timeout.
  - New handler `run_compute_analytics_from_csv_job` thin wrapper around
    `run_csv_strategy_analytics`.

### TypeScript surface (Tasks 7-11)

- `src/lib/types.ts`:
  - `computation_status` union for `strategy_analytics` extended with
    `"complete_with_warnings"` as a first-class DB status (not a UI-only state).
  - `AnalyticsDataQualityFlags` gets optional `csv_source?: boolean`.
- `SyncProgress.tsx` exhaustive switch updated; jsdoc refreshed per #274 (no
  stale "complete_with_warnings has no DB representation" narrative).
- `parseDailyReturnsSeries()` helper at `csv-finalize/route.ts` boundary:
  - Array shape, ≤5000 rows.
  - `YYYY-MM-DD` regex per date.
  - Finite-number per value.
  - **Duplicate-date guard at the route boundary** per #274 — returns 400
    `CSV_INVALID_FORMAT` instead of letting the UNIQUE constraint throw 23505
    as a 500 from the RPC.
- Persist call: legacy `csv-finalize` path fires `persist_csv_daily_returns`
  RPC after `applyCsvMetadataUpdate`. Persist error → 500 `CSV_PERSIST_FAIL`
  (hard-fail because `wizard_session_id` is unique-claimed; without persistence
  the worker loops on "Insufficient CSV history").
- Unified-backbone path mirror: `unifiedCsvFinalizeHandler` signature
  **explicitly** extended with `dailyReturnsSeries: CsvDailyReturnRow[]` (no
  closure capture — code-review guidance).
- `after()` enqueue on **both** paths, gated by `USE_COMPUTE_JOBS_QUEUE`.
  Non-blocking — failure logs a warning but never 500s. `@audit-skip` pragmas
  on both `enqueue_compute_job` call sites per #275 (compute_jobs is internal
  worker scheduling; user intent is captured by upstream `finalize_csv_strategy`
  + `persist_csv_daily_returns`).
